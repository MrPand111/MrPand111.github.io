关于搭建阿里云服务器的踩坑记录

发表于 2020-02-07 分类于 服务端 阅读次数：

昨天趁着疫情期间的活动，搞了一个半年的2核4G云服务器

1.基本配置

  1. 购买了一个服务器业务后，就会分配你一个实例（我选择的windows server操作系统，windows系统内存占比会高一些但操作相对简单，后面就以此操作系统讲解）。从【控制台】中找到【实例】，就可以进行管理了。

  2. 我们首先要修改的就是密码。第一个是【实例密码】，这个密码和账户密码的概念差不多，登录这个云服务器的系统是需要用户名（默认用户名Administrator）和密码。第二个是【远程连接密码】，这个密码是远程连接时需要输入的。修改完密码后记得重启下服务器。

2.远程连接

  1. 点击【远程连接】，这里有Workbench和VNC两种方式，workbench需要安全组开放几个ip，所以我们直接选择VNC的方式远程连接。

  2. 这时会跳出一个【管理终端】的页面，按要求输入【远程连接密码】后窗口就会变成windows开机后的界面，提示【Ctrl+Alt+Delete】解锁，这里千万不要去按组合键，因为你会打开任务管理器！在管理终端页面的左上角有【发送远程命令】的按钮，选择发送【Ctrl+Alt+Delete】即可。

  3. 这里会显示用户名为【Administrator】，输入的密码即为前面修改过的【实例密码】。到这里就已经进入操作系统的桌面嘞，因为是windows版，所以看着还是非常亲切而眼熟的。

  4. 通过网页操作的太卡，我们选择用windows自带的【远程桌面连接】（在windows左下角的搜索框直接搜）。第一项【计算机】填写我们云服务器的【公网IP】，第二项用户名填写【Administrator】（想要修改用户名点击【显示选项】），之后就可以点连接了，后面会提示输入凭据，这里填写【实例密码】。激动人心的时候又到了，再一次进入了windows的桌面！

3.搭配环境

  1. 虽然此时的云服务器上可以直接安装MySQL、nginx等环境或应用，但是我为了简化管理操作，安装了第三方管理软件————宝塔。在服务器的桌面上的IE中搜索下载【宝塔windows】，安装好后启动会有提示需要开放几个端口（比如运行宝塔操作面板的网页需要8888端口）。

  2. 我们返回到第一步【基本配置】时的控制台，找到【安全组】，安全组列表中有一个默认的安全组，我们对它进行【配置规则】，把宝塔提示的几个端口依次添加进去即可，这里提供一个模板。

    端口范围 授权对象 描述
    8888/8888 0.0.0.0/0 宝塔（随便写）

  3. 现在我们复制宝塔面板的网址链接到本机（注意是自己的电脑，云服务器上的辣鸡IE打不开面板的）的浏览器（建议chrome）上，用户名和密码在服务器上的宝塔应用中复制即可。

  4. 进入宝塔后会建议你初始化安装一些环境，我选择的是nginx套。（PS：一开始选择的IIS，毕竟都属于微软，但是后面建立站点跑不起来，原因不明中）

  5. 有了环境后我们就可以建立站点了，在【网站】中【添加站点】，第一个域名就是【公网IP】，后面的配置项看个人需求。建立完成后，去【文件】下找到对应的目录，删除初始化的几个文件，上传自己的网站文件即可。

第一次研究服务器，感觉自己有很多要学的，后面会不断学习丰富这篇博客的内容
